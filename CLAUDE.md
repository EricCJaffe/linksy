# CLAUDE.md — Linksy

AI-powered community resource search and referral platform (Clay County, FL). Natural language search → vector similarity + LLM → matched local providers. Multi-tenant SaaS: Site (Impact Works) → Tenants (regions) → Providers → Locations. Domain tables prefixed `linksy_*`.

## Stack

Next.js 14 App Router, TypeScript strict, Supabase (PostgreSQL + pgvector + PostGIS), OpenAI (`text-embedding-3-small` + `gpt-4o-mini`), Tailwind + shadcn/ui, React Query v5, React Hook Form + Zod, Vercel deploy.

## Commands

```bash
npm run type-check       # tsc --noEmit
npm run lint             # ESLint
npm run build            # Production build — all three MUST pass before commit
npm run test:run         # Vitest single run
npm run types:generate   # Supabase types → lib/types/database.ts
```

## Critical Rules (from real failures)

- **Pre-commit gate:** Run `type-check` + `lint` + `build` before every commit. CI will reject otherwise.
- **Supabase client choice matters:** `createClient()` respects RLS, `createServiceClient()` bypasses it. Using the wrong one causes silent data leaks or silent permission failures. Check `lib/supabase/server.ts`.
- **Column drift:** The `remote_schema` migration dropped columns (`assigned_to`, `linksy_ticket_events` table, etc). Always verify columns exist in `lib/types/database.ts` or migrations before referencing them in queries.
- **Never commit `SUPABASE_SERVICE_ROLE_KEY` or `OPENAI_API_KEY`** — server-only secrets.
- **Ticket numbers:** Format `R-<sequence>-<suffix>` (e.g. `R-2001-07`), generated by PG sequence + RPC. Don't generate in application code.
- **Commits:** Conventional Commits (`feat(scope):`, `fix(scope):`)

## Auth & Access Model

- **Roles:** `site_admin` > `tenant_admin` > `user` (in `users.role`); `admin`/`member` (in `tenant_users`); `admin`/`user` (in `linksy_provider_contacts.provider_role`)
- **Provider access:** Checked via `linksy_user_can_access_provider()` RPC — includes parent-admin inheritance
- **RLS:** Enforced on all tenant-scoped tables. `linksy_provider_contacts` has RLS disabled (auth at API layer)
- **Middleware** (`middleware.ts`): Protects `/dashboard/*`, rate limits, CSRF on mutations

## Project-Specific Conventions

- Use `cn()` from `lib/utils` for conditional Tailwind classes
- Data fetching: React Query hooks wrapping `fetch()` — add `staleTime`/`gcTime` to all `useQuery` calls
- Module system is data-driven (`modules` + `tenant_modules` tables), not env flags
- API routes: lowercase-dash naming
