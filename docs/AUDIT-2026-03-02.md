# Part 1: Code Review Findings

Reviewed 60+ files across 4 layers. 30+ issues found.

## CRITICAL — Fix Before Next Deploy ✅ ALL RESOLVED 2026-03-03

- [x] **XSS: Unsanitized `dangerouslySetInnerHTML`** — Added `isomorphic-dompurify` to `rich-text-display.tsx` and `rich-text-editor.tsx`.
- [x] **Missing `/api/invitations/accept` endpoint** — Created `app/api/invitations/accept/route.ts`.
- [x] **Open redirect in `/api/auth/callback`** — Added `safeRedirectPath()` to both callback routes and login form.
- [x] **Race condition in ticket numbering** — Created PG sequence + `linksy_next_ticket_number()` RPC.

## HIGH — Fix Soon ✅ ALL RESOLVED 2026-03-03

- [x] **OpenAI API calls missing error handling** — Added try/catch around embedding call with 503 fallback.
- [x] **Hardcoded SITE_ID in multiple routes** — Replaced with `process.env.LINKSY_SITE_ID` (with fallback).
- [x] **Provider API bypasses RLS with no tenant filter** — Switched to `createClient()` (respects RLS) + optional `tenant_id` param.
- [x] **Open redirect in login form** — Added `safeRedirectPath()` validation.
- [x] **Non-admin users can set `is_private` on comments** — Server now enforces `is_private` only for `site_admin`.
- [x] **Merge operation has no transaction/rollback** — Changed to fail-fast: returns error on first sub-operation failure.

## MEDIUM — Address When Possible (10 of 11 resolved)

- [x] **Crisis keyword test endpoint has no auth** — Added `requireAuth()` to `app/api/crisis-keywords/test/route.ts`. RESOLVED 2026-03-03
- [x] **`setTimeout` not cleaned up in find-help** — Added `useRef` + `clearTimeout` on unmount. RESOLVED 2026-03-03
- [x] **Search bar missing AbortController** — Added `AbortController` with cleanup on unmount/query change. RESOLVED 2026-03-03
- [x] **Notification subscription not tenant-scoped** — Scoped realtime subscription to `user_id` filter. RESOLVED 2026-03-03
- [ ] **In-memory rate limiter ineffective on Vercel** — `lib/utils/rate-limit.ts` stores state in-memory per instance. On multi-instance Vercel, limits don't work. **Use Upstash Redis for production.**
- [x] **Activity logging uses browser client** — Switched `lib/utils/activity.ts` from browser `createClient()` to server-side `fetch('/api/audit-logs')`. RESOLVED 2026-03-03
- [x] **`parseInt` NaN not handled** — Added `|| defaultValue` fallback across 6 route files. RESOLVED 2026-03-03
- [x] **Unsafe `any` types in hooks** — Added `TenantMembershipRow` interface to `useCurrentTenant.ts`, typed provider/category params in `find-help/page.tsx`. RESOLVED 2026-03-03
- [x] **Missing staleTime/gcTime on queries** — Added cache config to 11 queries across 8 hook files. RESOLVED 2026-03-03
- [x] **Error response info disclosure** — Replaced `validation.error.flatten()` with generic message in 4 routes. RESOLVED 2026-03-03
- [x] **CSRF allows `http://` origin in production** — Gated `http://` origin to `NODE_ENV === 'development'` only. RESOLVED 2026-03-03

## RLS Security Audit (Database Layer)

| Severity | Table | Issue |
|----------|-------|-------|
| **HIGH** | `linksy_provider_contacts` | RLS disabled entirely. Auth handled at API layer. Risk: any misuse of non-service-client could expose all contacts. |
| **HIGH** | `linksy_provider_notes` | `is_private` not enforced at RLS level. Private notes could leak to non-admin provider staff if application filtering is bypassed. |
| **MEDIUM** | `linksy_tickets` | No client-view policy. Clients who submitted referrals via widget cannot query their own ticket status. |
| **MEDIUM** | `linksy_call_logs` | Overly permissive: any authenticated user can manage any call log, not just their own provider's. |
| **MEDIUM** | `linksy_custom_fields` | Any authenticated user can manage any provider's custom fields. Should be scoped like `linksy_host_custom_fields`. |
| **MEDIUM** | `linksy_surveys` | Unrestricted UPDATE: any authenticated user can modify any survey result. |
| **LOW** | `linksy_search_sessions` | Anon update policy has no row-level filter. One session could theoretically modify another. |

### Database Recommendations

1. Re-enable RLS on `linksy_provider_contacts` with proper provider-scoped policies
2. Add `is_private = false` condition to `linksy_provider_notes` RLS for non-admin reads
3. Add client-email-based ticket view policy for `linksy_tickets`
4. Restrict `linksy_call_logs` to provider contacts for their own provider's tickets
5. Scope `linksy_custom_fields` to provider contacts (match `linksy_host_custom_fields` pattern)
